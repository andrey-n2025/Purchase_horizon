# Purchase_horizon
Модель прогнозирования вероятности покупки клиента в ближайшие 90 дней для оптимизации рассылок.

## Цель проекта
Разработать модель, которая по историческим данным о покупках и взаимодействии с рассылками предсказывает **вероятность совершения покупки клиентом в течение следующих 90 дней**.
Это позволитт:
- Фокусировать рассылки на "тёплых" клиентах
- Снизить количество спама
- Экономить маркетинговый бюджет на расссылки

## Данные
- `apparel-purchases.csv` — история покупок
- `apparel-messages.csv` — история рассылок
- `apparel-target_binary.csv` — целевой признак (сильный дисбаланс. положительный класс около 2%)

## Основные этапы проекта

1. Загрузка и EDA — анализ данных, выявление закономерностей
2. Предобработка — очистка, приведение типов, удаление дубликатов
3. Feature Engineering — RFM, поведенческие признаки по окнам (30/60/90 дней)
4. Моделирование — сравнение LogisticRegression, XGBoost, LightGBM
5. Оптимизация — RandomizedSearchCV, подбор порога
6. Анализ важности признаков

## Результаты модели
Лучшая модель: Logistic Regression + SMOTE
Финальные метрики на тестовой выборке:

ROC-AUC: 0.748
PR-AUC: 0.0618
Recall: 0.219 (при оптимальном пороге)
Precision: 0.088

**Ключевые признаки:**

Активность за последние 30 дней (открытия, клики)
давность последнего взаимодействия
Предпочтения категорий товаров
